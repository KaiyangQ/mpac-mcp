## 串成因果链

让 Explore subagent 先扫一遍代码地图,核心三个文件:

* `mpac-package/src/mpac_protocol/core/coordinator.py:1673` — `_detect_scope_overlaps` 只过滤 `intent_id == new_intent.intent_id`,**没过滤 `principal_id`**
* `mpac-mcp/src/mpac_mcp/relay.py:103-135` — system prompt 要求 Claude 按 `check_overlap → announce_intent → write → withdraw` 顺序操作
* `mpac-mcp/src/mpac_mcp/relay.py:235-264` — 子进程 exit-non-zero 时 relay **没有任何 intent 清理逻辑**

发现一条线索:

```
1. Dave 让 Claude "改 db.py 加 1000 行 hello world"
2. claude -p 子进程 announce_intent("notes_app/db.py") → intent A 注册成 ACTIVE
3. claude -p 试着 write_project_file 1000 行 print → Anthropic API 内容过滤拦截
4. 子进程 exit 1,relay 把错误转成字符串返回聊天框 (relay.py:259)
5. ❌ 没有 withdraw_intent 调用 → intent A 永远 ACTIVE
6. Dave 重发同一条 prompt,新 claude -p 启动
7. announce_intent 第二次 → intent B 注册成 ACTIVE,principal_id 跟 A 一样
8. _detect_scope_overlaps 比较 A vs B:同文件 + 不同 intent_id → 创建 Conflict
9. 但是 Conflict.principal_a == Conflict.principal_b == "agent:user-3"
10. UI 渲染:"Dave's Claude ↔ Dave's Claude"
```

3 个 bug 是一条因果链:

```
内容过滤 (Bug C) → 子进程死前留 orphan (无清理) → 同 principal 自冲突 (Bug B)
```

Bug A 是另一回事:Bob 的 Claude 调 `check_overlap()`(被动读)看到 Alice/Dave 在编辑 → 直接回话「我看到冲突了,问你怎么办」 → 不调 `announce_intent` → server 端没创建涉及 Bob 的 Conflict → 面板显示空。**这是 by design 的 UX 鸿沟**:面板冲突 ≠ Claude 主观感知冲突。

## (2c) announce 时 auto-supersede 同 principal 重叠 intent

[mpac-package/src/mpac_protocol/core/coordinator.py:859](mpac-package/src/mpac_protocol/core/coordinator.py:859) — `_handle_intent_announce` 在注册新 intent 之前,把同 principal 重叠的旧 ACTIVE intent 转成 WITHDRAWN,顺便调 `_cascade_intent_termination` + `_check_auto_dismiss`,跟正常 withdraw 路径一致:

```python
for other in list(self.intents.values()):
    if other.intent_id == intent.intent_id: continue
    if other.principal_id != intent.principal_id: continue
    if other.state_machine.current_state != IntentState.ACTIVE: continue
    if not scope_overlap(other.scope, intent.scope): continue
    other.state_machine.transition("WITHDRAWN")
    superseded_ids.append(other.intent_id)
for sid in superseded_ids:
    responses.extend(self._cascade_intent_termination(sid))

self.intents[intent.intent_id] = intent
responses.extend(self._detect_scope_overlaps(intent))
if superseded_ids:
    responses.extend(self._check_auto_dismiss())
```
