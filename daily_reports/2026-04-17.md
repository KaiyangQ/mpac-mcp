## Step 1：shadcn/ui 接入（手动 init，避开 globals.css 冲突）

`shadcn init` 默认会重写 `src/app/globals.css`——但我们的 globals.css 里已经有精心调过的 GitHub-Dark 配色（`--bg-primary: #0d1117`、`--accent: #58a6ff` 等），而且 Tailwind v4 用的是 CSS-first `@theme inline` 写法（没有 `tailwind.config.js`）。如果让 init 覆盖会把之前 Phase A 的主题全冲掉。

所以改走**手动 init** 路径：

1. 手写 [web-app/app/components.json](web-app/app/components.json)：`baseColor=zinc`、`cssVariables=true`、`tsx=true`、`iconLibrary=lucide`，aliases 指向 `@/components`、`@/lib/utils`、`@/components/ui`
2. 手写 [src/lib/utils.ts](web-app/app/src/lib/utils.ts) 的 `cn()` 工具函数（`twMerge(clsx(...))`）
3. 在 globals.css 里**增量追加** shadcn 需要的 CSS 变量，保留原有变量
4. 然后用 `npx shadcn@latest add button dialog command sonner tooltip tabs breadcrumb kbd scroll-area input alert`——`add` 命令不会碰 globals.css

### globals.css 里最 tricky 的一处：`--accent` 命名冲突

GitHub-Dark 主题里 `--accent: #58a6ff`（主色蓝，用于链接 / focus 描边）；shadcn 的 `--accent` 是**悬浮背景色**（command item hover 时的 bg）。同一个变量名，两套语义。

解决：**不把 shadcn 的 `--accent` 暴露成根 CSS 变量**，而是在 `@theme inline` 块里直接写 `--color-accent: var(--bg-tertiary)`——这样 shadcn 组件用 `bg-accent` class 时解析到 `--color-accent` → `#1c2128`（我们定义的 tertiary bg），同时原来所有 `text-[var(--accent)]` / `border-[var(--accent)]` 引用 `#58a6ff` 的代码照常工作。

其他 shadcn tokens（`--color-background` / `--color-foreground` / `--color-card` / `--color-muted` 等）都类似做了"映射到现有 GitHub-Dark 变量"的处理。还加了 `@custom-variant dark (&:is(.dark *));` 和 `import "tw-animate-css";`（shadcn dialog 的 fade 动画依赖）。

结果：11 个 shadcn primitives 安装后，外观和原来的 GitHub-Dark 主题无缝融合，连一处重新调色都没做。

## Step 4：⌘K 命令面板（demo-worthy 的一步）

新建 [src/components/command-palette.tsx](web-app/app/src/components/command-palette.tsx)（~170 行）：

### 结构

用 shadcn `<CommandDialog>` + `<CommandInput>` + `<CommandList>` + `<CommandEmpty>` + `<CommandGroup>` + `<CommandItem>` + `<CommandSeparator>`（这些来自 `cmdk` 库，shadcn 做了一层样式包装）。

三个 CommandGroup：
1. **Actions**（始终显示）：Invite collaborator（仅 owner 时显示） / Go to projects list / Sign out
2. **MPAC**（仅在有 active intent 或 pending conflict 时显示）：Yield current intent / Acknowledge conflict
3. **Files**：flatten 文件树后显示所有文件路径，onSelect → onJumpToFile

### 全局键盘绑定

```tsx
useEffect(() => {
  const handler = (e: KeyboardEvent) => {
    if (e.key === "k" && (e.metaKey || e.ctrlKey)) {
      e.preventDefault();
      onOpenChange(!open);
    }
  };
  window.addEventListener("keydown", handler);
  return () => window.removeEventListener("keydown", handler);
}, [open, onOpenChange]);
```

Cmd/Ctrl+K 切换 open 状态。`e.preventDefault()` 防止浏览器地址栏 focus（Chrome 默认绑定）。

### 关闭 → 动作的竞态处理

Radix Dialog 关闭时有一个 fade-out 动画（~200ms）。如果 palette 的 `onSelect` 直接先执行 action（比如打开 Invite Modal）再 close palette，两个 Dialog 会短时间叠加——Radix 内部 portal 栈可能搞出奇怪的 focus trap。

解决：写一个 `run(fn)` 包装器，**先 close palette，再 setTimeout 0 ms 后执行 action**：

```tsx
const run = (fn: () => void) => {
  onOpenChange(false);
  setTimeout(fn, 0);
};
```

`setTimeout(fn, 0)` 让 React 把关闭的 state update flush 到 DOM 后再触发 action。验证过 palette → invite modal 的 handoff 干净（测试时看到 palette 完全消失之后 Invite dialog 才 mount）。

## 踩到的坑

1. **`shadcn init` 会默认重写 globals.css**——如果直接跑会把之前 Phase A 调了一整天的 GitHub-Dark 配色冲掉。解决：完全绕开 init，手写 components.json + utils.ts，然后只用 `shadcn add`（它不碰 globals.css）。以后若需要添加新 primitive 直接 `add` 就行。

2. **Tailwind v4 的 `@theme inline` 变量名空间**：shadcn 官方示例是用 `@theme { --color-primary: ...; }` 这种形式（隐式 inline-variables），但因为我们已经有一套现成的 `--bg-primary` / `--accent` 等变量在 `:root`，得用 `@theme inline { --color-background: var(--bg-primary); }` 的**引用**形式才能共存。官方文档这块不够清楚，是在 `node_modules/next/dist/docs/` 和 Tailwind v4 docs 之间来回翻才想明白。

3. **`--accent` 语义冲突**：GitHub-Dark 的 `--accent` = 蓝色链接色，shadcn 的 `--accent` = hover 灰底。解决：shadcn 的 accent 只在 `@theme inline` 里映射（`--color-accent: var(--bg-tertiary)`），不暴露到 `:root`——两套语义各用各的 variable name。

4. **Next.js 16 client component 里的 `params` 是 Promise**：原先以为直接 destructure，结果 runtime 报错。正确写法是 `const { code } = use(params);`（`use()` 从 `'react'` import）。邀请页面第一次跑报错就是因为这个。

5. **Radix Dialog 关闭动画期间 DOM 还在**：palette → invite modal 的 handoff 如果不用 `setTimeout(fn, 0)`，两个 Dialog 会叠加一瞬间，导致第二个的 focus trap 不生效（用户按 Enter 会落到 palette 的隐藏 input 上）。

6. **lucide-react 图标默认 size 24px**——直接塞进 `<Button size="sm">` 看起来太大。统一加 `className="size-3.5"` 或 `size-4`。workspace 里的图标基本都是 `size-3.5`（14px）以保持 compact 感。

### 🐛 发现一个影响全站的 Tailwind v4 bug

Split-screen 第一版上线后，两边 content 视觉上不对称——左 panel 贴左边、右 form 居中。给 grid 加 `lg:max-w-7xl lg:mx-auto` 完全没效果。改了几轮都不对。

最后用 `getMatchedCSSRules` 挨个跑：登录页那个 grid 元素，**只有 `* { margin: 0 }` 在匹配**，Tailwind 所有 `mx-auto` / `m-*` / `p-*` utility 都没应用上。

根因：`globals.css` 里那条老派 reset：

```css
* {
  box-sizing: border-box;
  margin: 0;
  padding: 0;    /* ← 罪魁 */
}
```

**这条规则不在任何 `@layer` 里面**。Tailwind v4 的所有 utility 都在 `@layer utilities` 中。CSS cascade layer 规则：**unlayered CSS 永远赢过 layered CSS，不管 specificity 谁高**。结果：整个 codebase 里凡是 `m-*`/`p-*` utility 全部被 `* { margin: 0 }` 默默 zeroed，只是其他页面靠 `flex` / `gap-*` / padding utility 少，没暴露。

Fix：

```css
*, *::before, *::after {
  box-sizing: border-box;
}
```

只留 `box-sizing`，margin/padding 交给 Tailwind v4 preflight。一行删除，整个 app 的 spacing 全部复活——包括之前我以为已经"生效"了但实际被覆盖的地方。
