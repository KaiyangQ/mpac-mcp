## 改动明细

### `5e0ebc7` mpac 0.2.3: attribute-chain resolution + dependency_detail

**两块改动，一次 commit**。

#### Part A：扫描器 attribute-chain 解析

0.2.2 的 `_extract_imports_detailed` 看到 `import X` 就标 `None`（通配）—— 因为它不知道下游会怎么用 X。

0.2.3 加了一个 **AST attribute-chain pass**：对每个 `import X` / `import X as Y` 绑定，遍历 AST 收集所有 `Attribute(value=Name(alias))` 节点，把访问过的属性名搜集起来。

关键决策：**什么时候退回通配**？
- 任一 `Name(alias)` 节点的 parent **不是** `Attribute.value`（比如 `x = utils`、`return utils`、`isinstance(m, utils)`）→ 该 alias 通配
- `import pkg.sub` 这种带点的 → 通配（区分"submodule vs attribute"需要模块图）
- 没有任何使用（`import utils` 然后没碰它）→ 通配（防动态 eval / getattr）

**实现亮点**：
- 临时在节点上挂 `_mpac_parent` 属性来反查父节点（遍历完也不清理，下次 `ast.parse` 是新树）
- 每个 alias 独立追踪（同一文件可能 `import utils as u` + `import other_mod`，互不影响）
- 嵌套作用域也算（函数体内的 `utils.foo` 照样被收集）

**覆盖测试**（8 个新 case）：
- `import utils` + `utils.foo()` + `utils.bar()` → `['utils.foo', 'utils.bar']`
- `import utils as u` + `u.foo()` → `['utils.foo']`（别名正确还原）
- `import utils` + `x = utils` + `utils.foo()` → `None`（bare use 一票否决）
- `import utils` + `from utils import bar` + `utils.foo()` + `bar()` → `['utils.bar', 'utils.foo']`（合并两种导入）
- `import pkg.sub` + `pkg.sub.foo()` → `None`（保守）
- `import utils` 无任何使用 → `None`（防动态访问）
- `utils.Class.method()` → `['utils.Class']`（记外层 attribute，不穿透）
- 函数体里的 `utils.foo()` → `['utils.foo']`（走整个 AST）

#### Part B：CONFLICT_REPORT payload 带上 dependency_detail

新增 `mpac_protocol.core.scope.compute_dependency_detail(a, b)`：

```python
{
  "ab": [{"file": "<b's file>", "symbols": [...] | None}, ...],
  "ba": [{"file": "<a's file>", "symbols": [...] | None}, ...]
}
```

- `ab` = a 的改动波及 b 的哪些文件（和 `principal_a` / `principal_b` 对齐）
- 每个文件的 `symbols` 要么是 **交集** `affects_symbols ∩ impact_symbols[file]`（精确）
- 要么是 `None`（退回文件级 —— 当一侧没声明 `affects_symbols`，或该 importer 是通配）

`coordinator._detect_scope_overlaps` 只在 `category == "dependency_breakage"` 时带上这个字段；`scope_overlap` 分支保持不变。

**5 个新测试** 覆盖：精确交集、importer 通配退化、Alice 未声明退化、对称方向（ab + ba 都有）、无冲突时返回空 dict（防御性）。

加上一个 coordinator 级别的断言：`CONFLICT_REPORT` payload 里**确实**带了 `dependency_detail`（不然前端就收不到）。

共 **13 个新测试**。

文件数：5 改；+415 / -26 行。

**如果用户反馈"冲突卡片里没看到符号"**：
1. 浏览器 DevTools → Network → WS → `CONFLICT_REPORT`，看 payload 里 `dependency_detail` 是否存在。**没有** = 冲突双方都没人声明 `symbols` → 退回笼统文案，**正确行为**。
2. `dependency_detail` 存在但 `symbols` 是 `null` → 一侧没声明（通常是浏览器侧），文案会说"a file imported by"。也是**正确行为**。
3. `dependency_detail` 存在且 `symbols` 有值但 UI 没显示 → 前端 bug，检查 `page.tsx` 的 `dependencyRows` 渲染逻辑
4. `dependency_detail` 就是不来 → 后端 scope.py / coordinator.py bug，跑单测 bisect
