## 架构：Web-app 才是文件的真主人

这件事**开工才发现**：之前在讨论方案的时候，我（和用户）都假设 "client 端算"就是 MCP 客户端在本地文件系统扫。真去看代码发现：

- Web-app 是 hub，项目文件**存在 SQLite 的 `ProjectFile.content` 列里**，不在 Lightsail 文件系统上
- MCP client (`mpac-mcp`) 只是 HTTP relay，列文件 / 读文件 / 写文件全部 POST 给 web-app
- 「client 端算」在这个架构里 = 「web-app 端算」（从 DB 拉 rows 喂给 scanner）

这个细节翻转了 analyzer 的设计：**不能**依赖 `Path.rglob`，得做成**源无关**（`Mapping[path, content]`）。顺手加了个 FS adapter（`scan_reverse_deps_from_dir`）留给将来的 CLI / ref-impl 场景。

## 改动明细

### `8d9ffc1` mpac 0.2.1: cross-file dependency-breakage detection

**问题**：v0.2.0 `scope_overlap(a, b)` 只对 `a.resources ∩ b.resources` 求交集。两个 agent 改不同文件但代码相关联 → 不报冲突。

**核心改动**：

1. **`mpac_protocol/analysis/import_index.py`（新增）**
   - `scan_reverse_deps(target_files, project_files: Mapping[str, str]) -> list[str]`
     —— 源无关：喂它 `{path: content}` 字典就行
   - `scan_reverse_deps_from_dir(target_files, project_root)` —— 文件系统便利包装
   - `collect_python_sources_from_dir(root)` —— 单独的 FS → mapping 辅助，跳过 `__pycache__ / .venv / node_modules / dist` 等
   - AST-based：`ast.parse` + walk `ast.Import` / `ast.ImportFrom`
   - 相对 import 解析（`from .sibling import x` 在 pkg 里会正确解析到 `pkg.sibling`）
   - 目标既支持单文件也支持 `__init__.py`（后者会匹配所有 `from pkg.sub import ...`）
   - **Fail-soft**：任一文件的 `SyntaxError / UnicodeDecodeError / OSError` 都吞掉，坏文件绝不影响检测

2. **`mpac_protocol/core/models.py` — `Scope` 加 `extensions` 字段**
   ```python
   @dataclass
   class Scope:
       ...
       extensions: Optional[Dict[str, Any]] = None  # NEW
   ```
   `to_dict` / `from_dict` 都透传。SPEC.md §15.2 原本就给 `extensions` 这个扩展位，字段是 `implementation-specific data` —— 我只是在用。

3. **`mpac_protocol/core/scope.py` 加 `scope_dependency_conflict(a, b)`**
   - **只检测跨文件依赖，不检测同文件**（那是 `scope_overlap` 的事）
   - 逻辑：`(a.impact & b.resources) | (b.impact & a.resources)` 非空 → True
   - `_scope_impact()` helper 从 `scope.extensions["impact"]` 拉数据，类型校验兼容任何脏输入

4. **`mpac_protocol/core/coordinator.py` — `_detect_scope_overlaps()`**
   两层检测链：
   ```python
   overlap = scope_overlap(new_intent.scope, other.scope)
   dep_conflict = (not overlap
                   and scope_dependency_conflict(new_intent.scope, other.scope))
   if not overlap and not dep_conflict:
       continue
   category = "scope_overlap" if overlap else "dependency_breakage"
   ```
   关键：`scope_overlap` **语义不变**（SPEC.md §15.2.1.1 的 MUST 规则）—— 同文件撞击继续是 `scope_overlap`。`dependency_breakage` 是**新增**的独立检测路径。

5. **测试**：`mpac-package/tests/test_dependency_breakage.py` — 21 个新测试
   - Scanner：直接 importer、package 内、相对 import、排除目标自身、语法错误文件不崩、路径归一化
   - FS adapter：`.venv` / `__pycache__` 等被跳过
   - `scope_dependency_conflict`：正反向撞击、无 impact 降级（0.2.0 兼容路径）
   - Scope 序列化：extensions 往返保留、老 envelope 反序列化不崩、未知字段静默丢
   - **Coordinator 端到端**：Alice 改 utils + Bob 改 main → `category=dependency_breakage`
   - **回归**：同文件撞击仍然是 `scope_overlap`；两人无关 + 无 impact → 无冲突

6. **`mpac-package/pyproject.toml`**：0.2.0 → 0.2.1

**文件数**：7 改（含 2 新增 + 1 测试）、742 行新增、8 行修改

## 现在是什么样子

```
 ┌───────────────────────────────────────────────────────────┐
 │  Alice:  POST /api/agent/intents  { files: [utils.py] }   │
 │                                                           │
 │                     ↓                                     │
 │                                                           │
 │  web-app/api/mpac_bridge.py :                             │
 │    compute_scope_impact(db, proj_id, ["utils.py"]):       │
 │      rows = SELECT path, content FROM project_files       │
 │                WHERE proj_id=? AND path LIKE '%.py'       │
 │      → { "utils.py": "...", "main.py": "..." }            │
 │    scan_reverse_deps(["utils.py"], rows)                  │
 │      → ast.parse every file                               │
 │      → main.py has `from utils import foo`                │
 │      → 反向索引命中 → return ["main.py"]                  │
 │                                                           │
 │                     ↓                                     │
 │                                                           │
 │  Scope(kind="file_set",                                   │
 │        resources=["utils.py"],                            │
 │        extensions={"impact": ["main.py"]})                │
 │                                                           │
 │                     ↓                                     │
 │                                                           │
 │  SessionCoordinator._detect_scope_overlaps :              │
 │    Bob 已在的 intent.scope.resources = ["main.py"]        │
 │    ∩ Alice.resources = {utils.py} ∩ {main.py} = ∅         │
 │    → scope_overlap = False                                │
 │    → scope_dependency_conflict 看 impact :                │
 │      Alice.impact ∩ Bob.resources                         │
 │        = {main.py} ∩ {main.py} = {main.py}                │
 │      → True                                               │
 │    → 发 CONFLICT_REPORT                                   │
 │      category = dependency_breakage                       │
 │                                                           │
 │                     ↓                                     │
 │                                                           │
 │  两个浏览器 / agent 都收到 WebSocket push                 │
 └───────────────────────────────────────────────────────────┘
```

**开发者门槛**：零。老的 `begin_task` 请求不用改任何参数，后端自动填 impact。浏览器端看到的 `CONFLICT_REPORT.payload.category` 字段多了一个可能值。
