### 2. v0.1.10 — Execution and Governance Closure

在 `v0.1.9` 基础上，对协议正文进行了一轮更细的规范级审查，发现剩余问题仍属于“闭合修订”而不是“功能扩展”，因此继续沿用补丁版本号，推进到 `v0.1.10` 而不是 `v0.2.0`。归档目录为 `version_history/v0.1.10_execution_governance_closure/`，包含：

- `SPEC_v0.1.10_2026-04-04.md`
- `MPAC_v0.1.10_Update_Record.md`
- `MPAC_v0.1.10_Spec_Changeset.md`

本轮修订的重点包括：

- **Profile matrix closure**：Core Profile session 明确只允许 `post_commit`；`pre_commit` 必须搭配 Governance Profile
- **Pre-commit semantics closure**：coordinator 的授权不再直接等同于 `COMMITTED`，真正进入 `COMMITTED` 仍需 proposer 执行后发出完成声明
- **Escalation authority closure**：并发 `RESOLUTION` 改为“按当前 authority phase 做 first-resolution-wins”，升级后优先尊重 `escalate_to` / 明确授权的 arbiter
- **Conflict auto-dismiss alignment**：将 `TRANSFERRED` 补入 conflict auto-dismiss 使用的 terminal intent 集合
- **Claim approval auditability**：Governance Profile 下 `INTENT_CLAIM_STATUS(approved)` 必须记录 `approved_by`
- **Decision registry cleanup**：移除没有状态机语义的 `RESOLUTION.decision = deferred`

这轮修订没有增加新的 message type，但显著收紧了协议的可互操作语义边界。

### 9. v0.1.11 — Example and Schema Alignment

在 v0.1.10 完成后，对 root spec 做了一轮系统性审查，发现一类文档与规范正文不一致的问题以及几处边界情况未显式定义的模糊点。这些问题不涉及新功能，归入 `v0.1.11` 作为 closure revision。

本轮修订共 8 项：

1. **Section 28 示例消息修正**：5 个示例（HELLO、INTENT_ANNOUNCE、OP_COMMIT、CONFLICT_REPORT、RESOLUTION）全部补齐 `sender.sender_instance_id`（Section 11.2 MUST 要求），`version` 从 `"0.1.0"` 更新为 `"0.1.11"`
2. **`SESSION_INFO` payload 表补字段**：新增 `identity_issuer`（Optional），对齐 §23.1.4 credential exchange 响应示例
3. **`SESSION_CLOSE` summary 对齐**：§14.5 的示例从简略版改为 §9.6.2 定义的详细结构（含 per-state breakdowns）
4. **`COORDINATOR_STATUS` 交叉引用修正**：`heartbeat_interval_sec` 的引用从 §14.3 改为 §14.7.5
5. **`OP_BATCH_COMMIT` pre-commit 消歧义**：新增规则——coordinator 用 `batch_id` 是否已注册区分首次提交 vs 完成声明
6. **`INTENT_UPDATE` scope 扩大检测**：新增 SHOULD 级规范——scope 严格扩大时 coordinator 应重新做 overlap 检测
7. **`GOODBYE` transfer disposition 机制**：明确 `intent_disposition: "transfer"` 时 coordinator 把 intent 转 `SUSPENDED`，走 `INTENT_CLAIM` 认领路径；同时在 §15.6.1 Intent State Transition Table 补入对应的 `ACTIVE → SUSPENDED` 行，防止该路径与 normative table 的 "未列出即非法" 规则矛盾
8. **Semantic Profile placeholder 标注**：§20.3 注明为 v0.1.x 占位，详细定义留待后续版本
9. **术语统一**：Conflict State Transition Table（§17.8.1）的 guard condition 中 "current conflict phase" 统一为 "current authority phase"，与 §18.4 concurrent resolution rule 对齐
10. **README 降调**：README 的 Current Coverage 说明从 "full v0.1.11 message surface in schema plus both reference implementations" 改为如实声明 "root spec is at v0.1.11; JSON Schema and both reference implementations currently track v0.1.10"
