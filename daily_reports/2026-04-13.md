### 1. 产品路线定义与评审

#### 问题

面临两个方向选择：
1. 先做一个垂直杀手级应用（AI 协作编程产品）
2. 把 MPAC 暴露成 MCP Server

#### 结论

两个方向不是二选一，而是合并成一条路：

- **产品切口**：AI 协作编程（垂直场景）
- **接入形态**：MCP Server（分发渠道）
- **技术内核**：MPAC（协调语义）

一句话：**让不同人的 coding agent 在同一个 repo 里先协调、再动手。**

#### 2.3 暴露的 10 个 MCP Tool

| Tool | 用途 | 背后映射 |
|------|------|---------|
| `who_is_working` | 查看当前 repo 内有哪些 agent 在工作 | SESSION_SUMMARY |
| `begin_task` | 开始任务，宣告 intent | INTENT_ANNOUNCE + conflict detection |
| `check_overlap` | 检查文件集合是否和别人冲突 | scope_overlap 比对 |
| `get_file_state` | 获取文件当前 state_ref 和内容 | FILE_READ sideband |
| `submit_change` | 单次提交尝试 | OP_COMMIT + optimistic concurrency |
| `yield_task` | 主动礼让，释放 scope | INTENT_WITHDRAW |
| `ack_conflict` | 确认或争议冲突 | CONFLICT_ACK |
| `escalate_conflict` | 升级冲突到仲裁者 | CONFLICT_ESCALATE |
| `resolve_conflict` | 仲裁者发布裁决 | RESOLUTION |
| `take_over_task` | 接管崩溃 agent 的 suspended intent | INTENT_CLAIM |

#### 2.4 bridge 层身份/角色支持

`LocalParticipantBridge` 支持通过环境变量切换身份：

| 环境变量 | 用途 |
|---------|------|
| `MPAC_AGENT_NAME` | agent 显示名称 |
| `MPAC_PRINCIPAL_ID` | 协议层身份标识 |
| `MPAC_AGENT_ROLES` | 角色（contributor / arbiter） |
| `MPAC_AGENT_CAPABILITIES` | 可选能力声明 |

bridge 缓存按 `workspace + principal_id + roles` 三元组区分，不同身份不会复用同一个 WebSocket 连接。

#### 2.5 smoke 测试设计

所有 smoke 脚本共享两个关键设计：

1. **隔离 scratch workspace**：每次测试把目标文件复制到临时目录，不污染主仓库
2. **临时 ephemeral sidecar**：smoke 启动自己的 sidecar，跑完自动 stop，不留孤儿进程

通过 `launch_ephemeral_sidecar()` 和 `stop_sidecar()` 实现。

### 4. 环境搭建与 Claude Code 集成

#### 4.1 安装 Python 3.12

系统 Python 3.9.6 无法安装 `mcp` 包（要求 ≥3.10）。

```bash
brew install python@3.12    # → Python 3.12.13
```

在 `mpac-mcp/.venv` 创建 venv：

```bash
/opt/homebrew/bin/python3.12 -m venv mpac-mcp/.venv
mpac-mcp/.venv/bin/pip install mcp websockets pytest
```

安装结果：`mcp 1.27.0` + `websockets 16.0`

#### 4.2 修复 mpac_protocol lazy import

**问题**：`mpac_protocol/__init__.py` 无条件 import `MPACAgent`，而 `MPACAgent` 依赖 `anthropic` SDK。MCP 桥接层不需要 `MPACAgent`，但 import chain 触发了 `ModuleNotFoundError`。

**修复**：

```python
# Before
from .agent import MPACAgent

# After
try:
    from .agent import MPACAgent
except ImportError:
    MPACAgent = None  # anthropic SDK only needed for full LLM-backed agent
```

不改变任何现有行为，只是让 `mpac_protocol` 在没有 `anthropic` 的环境下也能正常 import server 和 core 模块。

#### 4.3 安装 Claude Code CLI

```bash
npm install -g @anthropic-ai/claude-code    # → Claude Code 2.1.104
```

#### 4.4 MCP Server 注册

验证 FastMCP 加载和 tool 注册：

```
FastMCP import OK
MCP app name: mpac-coding
Tools registered: 10
  - ack_conflict, begin_task, check_overlap, escalate_conflict,
    get_file_state, resolve_conflict, submit_change, take_over_task,
    who_is_working, yield_task
```

用 `claude mcp add-json` 注册为 local scope：

```bash
claude mcp add-json mpac-coding '{...}' --scope local
```

验证连接：

```
$ claude mcp list
mpac-coding: ... ✓ Connected
```

#### 4.5 发现：项目级 .mcp.json 在 -p 模式下不稳定

| 注册方式 | `claude mcp list` | `-p` 模式 tool 可用性 |
|---------|:---:|:---:|
| `.mcp.json`（project scope） | ✓ Connected | 不稳定（有时找到有时找不到） |
| `claude mcp add-json`（local scope） | ✓ Connected | 稳定可用 |

**结论**：local scope 注册 + `--dangerously-skip-permissions` 是 `-p` 模式下可靠的组合。交互式 session 中用户手动批准一次即可。
