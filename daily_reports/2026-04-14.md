### 4. 关键架构讨论：Web 应用需要什么

和用户对话澄清了一个非常重要的问题：**mpac-mcp 现有代码能不能直接支撑"两个异地用户在同一个网页上协作改代码"**？

答案：**不能，差两层**：
1. **跨机器传输**：当前 sidecar 绑 `127.0.0.1`，端口从本地路径 hash，异地两人完全无法相遇
2. **共享文件存储**：sidecar 的 workspace 是本机磁盘一个目录，没有跨用户共享的概念

好消息：**差的两层都不重**。协议本身天生是 WebSocket 的（`MPACServer` 底层就是 `websockets.serve`），改远程模式代码量很小。真正的大头是前端 UI 本身。

### 5. Remote Coordinator 模式实现（下午）

这是 Day 2 代码层面最重的一块，见 commit `b2f6b9e`，跨 7 个文件，+824/−16。

**设计目标**：通过一个环境变量 `MPAC_COORDINATOR_URL` 让客户端从本地 sidecar 切换到任意远程 coordinator，对上层 MCP 客户端完全透明，**旧行为 100% 向后兼容**。

**改动摘要**：

| 文件 | 改动 |
|---|---|
| [config.py](mpac-mcp/src/mpac_mcp/config.py) | `BridgeConfig` 加 `uri_override` / `auth_token` / `session_id_pinned` 三个字段；`build_bridge_config` 解析 `MPAC_COORDINATOR_URL`（从 `/session/<id>` path 提取 session id）+ `MPAC_SESSION_ID` + `MPAC_COORDINATOR_TOKEN` |
| [coordinator_bridge.py](mpac-mcp/src/mpac_mcp/coordinator_bridge.py) | 新增 `_ws_connect_kwargs` 在所有 WebSocket 连接上注入 `Authorization: Bearer`；`ensure_sidecar` 在 remote 模式只 probe 不 spawn；`fetch_session_summary` 在未 pin 时放宽 session id 检查；`launch_ephemeral_sidecar` 拒绝 remote 模式 |
| [sidecar.py](mpac-mcp/src/mpac_mcp/sidecar.py) | 加 `--tls` hint，非 loopback bind 时打印 hosted 模式 banner（bind/session/workspace/client URL/auth 提示） |
| [tests/test_config.py](mpac-mcp/tests/test_config.py) | +6 remote 模式测试（URL path 解析、env 覆盖、未 pin 的 fallback、token、frozen dataclass、local 回归） |
| [smoke_remote.py](mpac-mcp/src/mpac_mcp/smoke_remote.py) | **新 smoke**：本地起一个 clean-env sidecar 扮演 hosted coordinator，客户端走 `MPAC_COORDINATOR_URL` 驱动 bridge，验证 `is_remote + begin_task + yield` 全链路 |
| [pyproject.toml](mpac-mcp/pyproject.toml) | +`mpac-mcp-smoke-remote` console script；`requires-python` 修成 `>=3.10` |
| [docs/API.md](mpac-mcp/docs/API.md) | 新增 "Remote coordinator mode" 段：客户端用法 + hosted 启动 + Caddy 反代 auth 示例 + smoke 用法 |

**三个关键设计决定**：
1. **URL 格式用 path (`wss://host/session/<id>`)** 而不是 query string——对 Caddy/nginx 反代更友好，可以按路径路由、限流、日志分桶
2. **Token 校验放反向代理，不改 `mpac` 核心**——客户端只负责发 `Authorization: Bearer <token>`，校验由部署时的 Caddy/nginx 做。这样 `mpac` 核心包保持纯协议、auth-free
3. **版本号保持 `0.1.0`**——既然还没发 PyPI 就不用 bump
