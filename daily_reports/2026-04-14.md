### 4. 新民会议：战略方向 pivot（关键）

下午和新民开了第二次会议，会议决定：

#### 产品策略
- **开源 + 付费双版本模式**：开源版本积累用户和社区，企业定制版提供高级功能
- **优先开发 MCP server 本地部署版本**，不购买 200+万/月的商业云服务
- **聚焦 coding 场景**，采用类似 **Overleaf** 的界面设计（左侧显示共同操作的文件）
- **发布节奏**：MCP server 完成后若可独立开源就先发布，不等垂直应用

#### 用例设计原则
- 选择**覆盖面广但深度适中**的例子
- 目标是让尽可能多的人达到购买冲动的 threshold
- 不要让少数人最大化满意

#### 用户增长路径
- MCP server 发布后建立 Slack/微信群
- 积累首批 100 个用户后再推付费版
- 需要拿下 1-2 个大客户作为标杆

#### 行动项（KAIYANG）
- 让 AI 列出论文所需的具体实验清单并评估可行性（本周）
- 开发 MCP server 本地部署版本（两周内）
- 设计 coding 场景的测试用例
- 开发类 Overleaf 界面的垂直应用（两周内）
- **保持每周至少 3-4 次的进度更新频率**（长期要求）

#### 关键认识

会议里"Overleaf 风格"的锚点改变了产品形态：**不是让用户装 VS Code 插件，而是一个开箱即用的 Web 应用**——打开 link 就能和别人/AI agent 协作编辑同一个 repo，MPAC 在后面做协调。左侧的"共同操作文件"就是显示"Alice 正在改 auth.py，Bob 在改 api.py"这种实时状态。
