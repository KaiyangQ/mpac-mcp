# 事件一：ssh mpac 挂了

想偷懒先 `ssh mpac 'git rev-parse HEAD'` 看 prod 上跑啥：

```
no such identity: /Users/kaiyangqian/Library/CloudStorage/OneDrive-.../Agent_talking/LightsailDefaultKey-us-west-2.pem: No such file or directory
ubuntu@184.32.168.112: Permission denied (publickey).
```

`~/.ssh/config` 里：

```
Host mpac mpac-web
  HostName 184.32.168.112
  User ubuntu
  IdentityFile ~/Library/CloudStorage/OneDrive-.../Agent_talking/LightsailDefaultKey-us-west-2.pem
  IdentitiesOnly yes
```

昨晚 daily report 里"仓库脱 OneDrive"只搬了 work tree（git 仓库本身），**`~/.ssh/config` 是独立文件，没动**，路径就这么悬空了。pem 也不在 OneDrive 老 Agent_talking 目录里 —— 应该是搬仓库那次连同从 OneDrive 拿出来过，但没明确放到一个固定位置。

## 找回来

KAIYANG 也忘了在哪。我 `find ~ -maxdepth 5 -iname 'LightsailDefaultKey*'` 顺便先看几个常见地方：

- `~/.ssh/` —— 没有
- OneDrive 老 Agent_talking —— 没有
- **`~/Downloads/LightsailDefaultKey-us-west-2.pem`** —— 找到了，1675 bytes，4 月 17 号下载，对得上第一次部署 Lightsail 那天

权限是 `0644`（默认下载权限），SSH 会因为"权限太开"拒用。

## 修

三步：

```bash
mv ~/Downloads/LightsailDefaultKey-us-west-2.pem ~/.ssh/
chmod 600 ~/.ssh/LightsailDefaultKey-us-west-2.pem
# Edit ~/.ssh/config: IdentityFile → ~/.ssh/LightsailDefaultKey-us-west-2.pem
```

`ssh mpac 'echo ok'` 通了。pem 现在在标准位置（`~/.ssh/`），脱 OneDrive 之后不会再悬空。

远端：

```bash

    git reset --hard origin/deploy && \
```
